# Feature Delivery Skill

## Purpose

Use this skill when implementing a feature from an intent file. This is the default, general-purpose delivery skill — use a more specific skill (React Form Skill, API Integration Skill, etc.) alongside this one when the work falls into a specialized category.

## Process

1. Read the intent file completely.
2. Identify the business outcome.
3. Identify acceptance criteria.
4. Inspect the repository before changing code.
5. Find existing patterns similar to the requested feature.
6. Create a small implementation plan.
7. Modify the smallest reasonable set of files.
8. Add or update tests.
9. Run relevant validation commands.
10. Produce delivery evidence.

## Do Not

- Do not invent requirements.
- Do not skip repo inspection.
- Do not introduce unnecessary dependencies.
- Do not rewrite unrelated code.
- Do not ignore acceptance criteria.
- Do not claim success without evidence.

## Required Output

Provide:

- What was built
- What files changed
- What tests were added
- What commands were run
- What acceptance criteria were satisfied
- What risks remain
