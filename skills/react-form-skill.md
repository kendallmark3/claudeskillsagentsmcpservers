# React Form Skill

## Purpose

Use this skill when creating or modifying a React form.

## Rules

1. Inspect the existing form patterns in the repo before creating new components.
2. Reuse the team's design system components.
3. Do not introduce a new form library unless the intent file explicitly requires it.
4. Keep form state predictable and easy to test.
5. Validate required fields before submit.
6. Display user-friendly validation messages.
7. Preserve accessibility labels, keyboard navigation, and screen reader support.
8. Add tests for happy path, validation errors, and submit failure.

## Required Steps

1. Read the intent file.
2. Locate similar forms in the repo.
3. Identify existing components, hooks, validators, and API clients.
4. Implement the form using existing conventions.
5. Add or update tests.
6. Run the relevant test command.
7. Document what changed.

## Evidence Required

At the end of the work, provide:

- Files changed
- Components reused
- Tests added
- Test command executed
- Any assumptions made
- Any gaps or follow-up work
